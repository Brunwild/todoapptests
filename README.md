## Список реализованных тестов
1. Покрыты CRUD операции (Create, Read, Update, Delete)
2. Учтены edge-cases JSONPlaceholder (успешные “фейковые” PUT/DELETE)
3. Проверены фильтры и UI состояния
4. Учтена отсутствие персистентности данных
5. Проверены race conditions из-за задержки API

## Инструкция по запуску
1. Скачать проект
2. Установить зависимости pip install -r requirements.txt
3. playwright install chromium
4. cd todoapptests\tests
5. pytest tests/ -v --html=reports/report.html --self-contained-html \\ Запустить тесты с сохранением результатов
6. allure generate reports/allure-results -o reports/allure-report --clean \\ Сгенерировать отчёт
7. allure open reports/allure-report \\ Открыть отчёт(не всегда работает, отчет генерируется в todoapptests\tests\reports\report.html)

### Параллельный запуск (по браузерам в разных процессах):
pytest -n 3 tests/ --asyncio-mode=auto

## Возникшие трудности
1. Конфликт Angular 16 и Node: 22.12.0, пришлось откатиться к 20 версии.
2. Трудности с созданием структуры с нуля, не лучшая идея сразу начать писать проверки, а потом структурировать проект. Переделал всё, основываясь на задуманной структуре проекта, прибегнув к вайбодингу, тесты виснут.
4. Проблема зависания решена, но теперь тесты могут фейлилться из-за того, что backend часто возвращает 500 ошибку при PUT-запросах, и начальный список задач не всегда успевает загрузиться. 
5. Проблема зависания не решается добавлением функции, которая создает новую задачу перед тестом. 
7. Запросы API замоканы, слишком много правок от нейронки.
8. Слишком много вайбкодинга, вроде всё работает, но немного потеряно восприятие как. 
9. Не всегда открывается страничка с репортом allure.
