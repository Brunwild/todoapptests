### Баг-репорт №1
Описание: Отсутствует валидация поля ввода при добавлении пустой задачи — задача не добавляется, сообщение об ошибке не показывается.

Severity: Minor
Priority: Medium
Компонент: UI / Добавление задачи
Окружение: win 11, Angular Todo App, браузер Chrome (последняя версия), JSONPlaceholder API

#### Шаги воспроизведения:
- Открыть главную страницу приложения.
- Поле ввода «What needs to be done?» оставить пустым.
- Нажать кнопку «Add».

#### Фактический результат:
Кнопка «Add» активна. После нажатия задача не добавляется. Сообщение об ошибке отсутствует, визуальной обратной связи нет.

#### Ожидаемый результат:
Либо кнопка «Add» должна быть неактивна (disabled) при пустом поле, либо при нажатии должно появляться сообщение («Текст задачи не может быть пустым» или аналогичное).

Дополнительная информация:
Тест-кейс №6 (негативный) не пройден. Пользователь введён в заблуждение — действие выполнено, но результат отсутствует.


#### Баг-репорт №2
Описание: При добавлении задачи с очень длинным текстом нарушается вёрстка: текст выходит за пределы контейнера, чекбокс уменьшается

Severity: Minor
Priority: Low
Компонент: UI / Стилизация списка задач
Окружение: win 11, Angular Todo App, браузер Chrome (последняя версия), JSONPlaceholder API

#### Шаги воспроизведения:
- В поле ввода ввести строку длиной более 120 символов (например, повторяющаяся буква «A»).
- Нажать кнопку «Add».

#### Фактический результат:
Текст задачи не переносится и не обрезается, «уезжает» за правую границу родительского блока. Чекбокс слева от текста визуально уменьшается в размере (сжимается).

#### Ожидаемый результат:
Текст должен переноситься на следующую строку (word-wrap, overflow-wrap) либо обрезаться с многоточием. Чекбокс должен сохранять стандартные размеры и пропорции.

Дополнительная информация:
Тест-кейс №7 (граничный случай) не пройден. Проблема влияет на читаемость и общее восприятие интерфейса, но критичности не представляет.

#### Баг-репорт №3
Описание: Отсутствует индикатор загрузки при добавлении задачи в условиях задержки ответа API

Severity: Minor
Priority: Medium
Компонент: UX / Взаимодействие с API
Окружение: win 11, Angular Todo App, браузер Chrome (последняя версия), JSONPlaceholder API, имитация медленного соединения (Throttling: Slow 3G или задержка API 1–2 секунды)

#### Шаги воспроизведения:
- С помощью инструментов разработчика установить замедление сети (например, «Slow 3G»).
- Ввести текст новой задачи и нажать «Add».
- Наблюдать за интерфейсом в промежутке между кликом и появлением задачи в списке.

#### Фактический результат:
После нажатия «Add» никакой визуальной обратной связи нет: нет спиннера, надписи «Сохранение...», disabled-состояния кнопки. Задача появляется в списке только после получения ответа от сервера (через 1–2 секунды). Пользователь может повторно нажать «Add», не понимая, что запрос уже отправлен.

#### Ожидаемый результат:
На время выполнения запроса к API должен отображаться индикатор загрузки (например, спиннер или текст «Добавление...»), кнопка «Add» должна стать неактивной до завершения операции.

Дополнительная информация:
Тест-кейс №8 (Edge-case, связанный с задержкой API) не пройден. Отсутствие индикатора ухудшает UX и может привести к дублированию задач при повторных кликах.
