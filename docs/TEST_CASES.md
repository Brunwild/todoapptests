### 1. Добавление новой задачи (позитивный)
#### Предусловия:
Открыта главная страница приложения, список задач загружен из API

#### Шаги:
- В поле ввода «What needs to be done?» ввести текст «Купить молоко»
- Нажать кнопку Add

#### Ожидаемый результат:
В списке задач появляется новая запись: «Купить молоко» в статусе Active (не выполнена). Поле ввода очищается.


### 2. Отметка задачи как выполненной (позитивный)
#### Предусловия:
В списке есть хотя бы одна активная задача (например, «delectus aut autem»)

#### Шаги:
- Нажать на чекбокс (квадратик) слева от выбранной задачи

#### Ожидаемый результат:
Чекбокс становится отмеченным (галочка), задача переходит в статус Completed. При фильтре «Completed» эта задача отображается, при «Active» скрывается


### 3. Редактирование текста задачи (позитивный)
#### Предусловия:
В списке есть задача с текстом «fugiat veniam minus».

#### Шаги:
- Нажать кнопку «Edit» рядом с этой задачей.
- В появившемся поле изменить текст на «Test № 3».
- Подтвердить редактирование (например, нажать Enter или кнопку Save).

#### Ожидаемый результат:
Текст задачи обновляется на «Новый текст задачи». Остальные атрибуты (статус выполнения) не меняются.


### 4. Удаление задачи (позитивный)
#### Предусловия:
В списке есть задача «quis ut nam facilis et officia qui».

#### Шаги:
- Нажать кнопку «Delete» рядом с этой задачей.

#### Ожидаемый результат:
Задача исчезает из списка. Количество задач уменьшается на 1.


### 5. Фильтрация активных задач (позитивный)
#### Предусловия:
В списке присутствуют как выполненные, так и невыполненные задачи (например, из предоставленного списка: 4 задачи выполнены, остальные активны).

#### Шаги:
- Нажать на фильтр «Active».

#### Ожидаемый результат:
Отображаются только задачи без галочки (статус Active). Выполненные задачи скрыты. Фильтр «Active» визуально активен.


### 6. Попытка добавления пустой задачи (негативный)
#### Предусловия:
Открыта главная страница.

#### Шаги:
- Оставить поле ввода пустым (или ввести пробелы).
- Нажать «Add».

#### Ожидаемый результат:
Задача не добавляется. Появляется сообщение об ошибке (например, «Текст задачи не может быть пустым»). Список задач не изменяется.


### 7. Добавление задачи с очень длинным текстом (граничный случай)
#### Предусловия:
Открыта главная страница.

#### Шаги:
- В поле ввода ввести строку из 500 символов (например, повторяющаяся буква «А»).
- Нажать «Add».

#### Ожидаемый результат:
Задача успешно добавляется. В списке текст отображается (возможно, с переносом или обрезкой, но не ломает вёрстку). Приложение не падает.


### 8. Поведение при задержке ответа API (Edge-case)
#### Предусловия:
Имитация медленного соединения (задержка ответа от JSONPlaceholder до 2 секунд).

#### Шаги:
- Добавить новую задачу (например, «Тест задержки»).
- Сразу после нажатия «Add» обратить внимание на интерфейс.

#### Ожидаемый результат:
На время отправки запроса появляется индикатор загрузки (спиннер или надпись «Сохранение...»). После получения ответа задача появляется в списке. При повторной операции задержка не приводит к дублированию задач.


### 9. Поведение при refresh страницы (API особенность)
#### Предусловия:
Приложение загружено, список отображает начальные задачи из API.

#### Шаги:
- Добавить новую задачу «Временная задача».
- Дождаться её появления в списке.
- Обновить страницу (F5).

#### Ожидаемый результат:
После перезагрузки добавленная задача «Временная задача» исчезает. Список возвращается к исходному состоянию (только задачи, полученные от API). Это соответствует поведению JSONPlaceholder, который не сохраняет изменения.


### 10. Переключение фильтров после выполнения задачи (комбинаторный)
#### Предусловия:
Включен фильтр «Active», видна только одна активная задача.

#### Шаги:
- Отметить эту активную задачу как выполненную (нажать чекбокс).

#### Ожидаемый результат:
Задача сразу пропадает из текущего списка (так как фильтр «Active» больше не должен её показывать). При переключении на фильтр «Completed» задача отображается в списке выполненных. При «All» — тоже отображается с галочкой.


### 11. Переключение Completed → Active (edge-case)
#### Предусловия:
В списке присутствует невыполненная задача

#### Шаги:
- Отметить задачу как выполненную
- Снять отметку

#### Ожидаемый результат:
- Статус возвращается в Active
- Фильтры обновляются корректно
